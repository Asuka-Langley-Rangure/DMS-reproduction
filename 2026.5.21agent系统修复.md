1. 主循环与总体控制流

去掉了 frozen stage-plan 主链路，runner 改回论文式主循环：
每轮 planner 直接基于当前 observation 产出 subtasks
runner 顺序执行 subtasks
subtask 结束后统一做 verifier / evaluator 判定
移除了对旧 current_stage_id / initial_stage_plan 路径的依赖。
修掉了 initial_stage_plan_record is not defined 的收尾崩溃，避免任务已完成但 TaskRunResult 封装时报错。
2. Actor 结束语义与 stop 机制

引入了 ActorStatus="stopped"，把 actor 的 heuristic brake 从伪装 completed 改成显式 stopped。
runner 现在区分：
actor 自身完成
actor 停手交给 verifier
whole-task evaluator 真正判成功
后续又把 AndroidWorld evaluator 提到最高优先级：
每个 subtask 后先查 task.is_successful(env)
成功则直接 whole-task success
否则再跑 LLM verifier
3. 去除 contact-only 特化，改成通用 form / subtask 逻辑

去掉了 contact-only 的 runner 控制流和 success override 依赖。
改成通用 form-filling / save-and-verify / result verification 规则。
planner / actor / runner 不再依赖 contact_form_context 这类 task-specific 后处理。
4. Planner：从单步 repair 到论文式多 subtask 分解

修复了 planner 经常只输出 1 个 subtask 的问题，现在支持并实际产出 1-5 个 subtasks。
强化了 generic planner prompt，让它更偏 round-level decomposition，而不是单步补丁式 replanning。
对 Contacts 这类任务加入了更强的 current observation grounding：
当前 screen 优先于 history
已在相关 app/workspace 内时，不应再规划 open-app 子任务
加入了 planner output veto：
如果当前 observation 已在 Dialer/Contacts，但 planner 还输出 Open the Phone app
runner 会判定 planner_subtask_conflicts_with_current_observation 并重规划
给 generic planner path 注入了 current-stage hint，尤其覆盖：
contacts/create-contact entry
contact editor/detail
app-already-open 场景
5. Actor 输入 grounding：索引契约与截图策略

在 actor prompt 中加了硬约束：
只能使用 Visible UI index table 里的合法 index
禁止抄 screenshot 上未出现在 table 里的数字
actor 使用的 labeled screenshot 改成 box-only：
保留红框
去掉数字
文本 index table 继续保留
planner / verifier 暂时不跟着改 screenshot 标注策略，先只改 actor 路径。
6. Actor 动作归一化

增加了 swipe -> scroll 归一化：
swipe up -> scroll up
swipe down -> scroll down
prompt 中明确写了：
不要输出 swipe
用 scroll
7. Actor 开头硬约束：显式 open-app goal 直接走 OpenAppAction

在 run_subtask() 开头接入了 direct-open fast path：
若当前 subtask 是显式 Open/Launch <App>
且当前不在目标 app 内
直接执行 OpenAppAction(app_name=...)
不走 LLM
该路径会生成完整 step artifact，不是静默执行。
成功后默认直接 stopped，交给外层 verifier/evaluator，而不是 actor 自己判完成。
8. Actor pointing correction / 目标元素吸附

重写了 _maybe_correct_pointing_payload(...) 的决策逻辑：
唯一语义匹配且 clickable -> 改合法 index
唯一语义匹配但 non-clickable 且有 bbox -> 改 x/y
多个匹配时：
clickable 子集唯一 -> 用该 index
无 clickable，但 exact text/content_description 唯一 -> 用该元素 x/y
否则不修，避免误吸附
坐标 fallback 改成基于“匹配到的目标元素”生成，不再依赖原始错误 index 指向的元素。
保留了 control-widget fallback，但放到新规则之后。
最小增强了 _extract_target_token(...)，更稳定支持：
Settings
Wi-Fi
Network & internet
一些联系人场景标签
9. Actor observation 时序稳定化

对高影响动作后的 stopped 路径加了 observation resample：
第一次 after_observation 正常抓取
命中 heuristic stop 时，再额外 capture_observation(...) 一次
第二次结果覆盖 step 的 after_observation 和 ActorRunResult.final_observation
在 artifact 中标记：
observation_attempt = 2
observation_resampled = true
final_after_resample = true
subtask_summary.md 中也增加了 Observation resampled: True/False
10. Editable field click 不再过早触发 verifier

调整了 _should_stop_for_external_verification(...)：
click / long_press 命中 is_editable=True 的字段时，不再 stop
input_text / keyboard_enter 后才允许 stop
这样 text-entry subtask 不会因为“先点一下输入框”就立刻交给 verifier。
11. Text-entry 动作恢复与 prompt 统一

修复了 {"action_type":"type","text":"Ahmed"} 这类失败：
先归一化 type -> input_text
在安全条件下自动补成：
{"action_type":"input_text","index":<resolved_editable_index>,"text":"Ahmed","clear_text":true}
补 index 的顺序是：
唯一 focused editable field
最近点击且仍可见的 editable field
唯一 visible editable field
统一并去重了 actor prompt 里的 text-entry 硬规则：
不要输出 type
文本录入直接用 input_text
不要只点字段不输入
First name / Last name 分开时输入对应部分
12. Verifier 结果语义与 evaluator 融合

VerifierResult 新增了 source 字段：
llm_verifier
androidworld_evaluator
heuristic
evaluator 成功时，runner 会合成标准 VerifierResult(status="success", source="androidworld_evaluator")
不再把 evaluator success 作为 verifier 外的一条特殊旁路；它现在是最高优先级的 verifier 来源。
13. Memory 记录与成功轨迹写入

memory event 现在会记录 verifier_source。
evaluator success 也会进入 memory event，因此 active memories 能收到成功反馈。
trajectory 写入条件收紧为：
verifier_result.status == "success"
verifier_result.memory_eligible == True
且 actor 轨迹没有 parse/execution error
evaluator-success 的 memory_eligible 改成本地规则：
len(actor_result.steps) > 1
且无 parse/execution error
14. Artifact / 日志可观测性增强

扩展了 round_summary.md：
每个 subtask 现在会汇总：
actor status
actor completion message
actor final action / reason
verifier status / reason
subtask_summary.md 增加了更多诊断信号：
form fill progress
observation resampled
text-entry success detected 等
这些改动让排查 planner / actor / verifier 分歧更直接。
